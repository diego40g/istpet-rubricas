A5. Simulación de Entrevista y Redacción de Historias de Usuario

📋 Descripción

Esta actividad tiene dos partes integradas. Primero simulas una entrevista semi-estructurada con el bibliotecario del Sistema de Biblioteca Digital para descubrir requisitos. Luego transformas los requisitos descubiertos en historias de usuario con criterios de aceptación Gherkin verificables.

🎯 Objetivos de Aprendizaje
  1. Aplicar técnicas de entrevista semi-estructurada usando los 5 tipos de preguntas.
  2. Transformar respuestas conversacionales en requisitos funcionales formales.
  3. Redactar historias de usuario en formato Como / Quiero / Para.
  4. Evaluar la calidad de una HU aplicando el criterio INVEST.
  5. Formular criterios de aceptación verificables en formato Gherkin.

Banco de Preguntas

# Tipo Pregunta formulada
1 Contexto ¿Puede describir su rutina diaria como bibliotecario?
2 Problema ¿Cuáles son los mayores inconvenientes con el sistema actual?
3 Problema ¿En qué tareas invierte más tiempo del que debería?
4 Solución ¿Cómo le gustaría gestionar los préstamos idealmente?
5 Prioridad Si pudiera mejorar una sola cosa del sistema actual, ¿cuál sería?

9 Problema ¿Qué pasa cuando un libro ya está prestado y otro usuario lo solicita?
10 Contexto ¿Cuántos usuarios simultáneos acceden al sistema en hora pico?

Reflexión sobre la Técnica

¿Qué funcionó bien?

  • Las preguntas de contexto permitieron entender el flujo real de trabajo antes de profundizar en necesidades específicas.
  • Las preguntas de problema revelaron puntos de dolor no evidentes, como la falta de alertas automáticas.
  • Las preguntas de validación fueron clave para evitar malentendidos; confirmar "¿3 días antes?" previno un requisito incorrecto.
  • Escuchar activamente (80/20) permitió descubrir la necesidad de reportes y multas, que no estaban en el brief inicial.

¿Qué se podría mejorar?

  • Usar una grabación (con permiso) para no perder detalles durante la transcripción.
  • Preparar un prototipo en papel para la siguiente sesión.
  • Incluir a un usuario final (lector) en una segunda entrevista para contrastar perspectivas.
Parte B — Historias de Usuario

HU-001: Búsqueda de Libros  |  Alta  |  Estimación: 3 puntos

Como usuario de la biblioteca,
quiero buscar libros por título, autor o categoría,
para encontrar rápidamente el material que necesito.

Evaluación INVEST

Criterio Cumple Justificación
Independent ✅ No depende de otras historias para ser implementada.
Negotiable ✅ Los filtros de búsqueda pueden ajustarse con el equipo.
Valuable ✅ Es la función principal que los usuarios utilizarán.
Estimable ✅ Funcionalidad estándar, estimable con claridad.
Small ✅ Alcance acotado a búsqueda y listado de resultados.
Testable ✅ Se puede verificar con casos de prueba de búsqueda.

Criterios de Aceptación

# CA-001: Búsqueda exitosa por título Dado que soy un usuario autenticado en la plataforma Cuando ingreso "El Principito" en el campo de búsqueda y presiono "Buscar" Entonces el sistema muestra una lista con todos los libros que contengan ese texto en el título Y cada resultado muestra título, autor, categoría y disponibilidad
# CA-002: Búsqueda por autor Dado que estoy en el buscador Cuando ingreso el nombre "García Márquez" como criterio de autor Entonces el sistema muestra todos los libros registrados de ese autor
# CA-003: Sin resultados Dado que realizo una búsqueda Cuando no existe ningún libro que coincida con el criterio ingresado Entonces el sistema muestra el mensaje "No se encontraron resultados para tu búsqueda"

HU-002: Préstamo de Libro  |  Alta  |  Estimación: 5 puntos

Como usuario registrado de la biblioteca,
quiero solicitar el préstamo de un libro disponible,
para poder leerlo durante el período permitido.

Evaluación INVEST

Criterio Cumple Justificación
Independent ✅ Depende de HU-001 solo funcionalmente, no técnicamente.
Negotiable ✅ La duración del préstamo es negociable con la biblioteca.
Valuable ✅ Es el proceso central del negocio.
Estimable ✅ Flujo bien definido, estimable.
Small ✅ Cubre únicamente la acción de solicitar el préstamo.
Testable ✅ Verificable con casos de prueba de disponibilidad y estado.

Criterios de Aceptación

# CA-001: Préstamo exitoso Dado que soy un usuario autenticado sin préstamos vencidos Y el libro "Cien años de soledad" está disponible Cuando hago clic en "Solicitar préstamo" Entonces el sistema registra el préstamo con fecha de inicio y fecha de vencimiento (14 días) Y el estado del libro cambia a "Prestado" Y recibo un email de confirmación con los detalles del préstamo
# CA-002: Libro no disponible Dado que el libro seleccionado ya está prestado Cuando intento solicitar el préstamo Entonces el sistema muestra "Este libro no está disponible actualmente" Y me ofrece la opción de reservarlo
# CA-003: Usuario con préstamos vencidos Dado que tengo un préstamo con fecha de vencimiento superada Cuando intento solicitar un nuevo préstamo Entonces el sistema muestra "Tienes préstamos vencidos. Debes devolver los libros antes de solicitar nuevos."

HU-003: Devolución de Libro  |  Alta  |  Estimación: 3 puntos

Como usuario con un préstamo activo,
quiero registrar la devolución de un libro,
para liberar mi cuenta y permitir que otros usuarios lo soliciten.

Evaluación INVEST

Criterio Cumple Justificación
Independent ✅ Independiente, solo requiere que exista un préstamo activo.
Negotiable ✅ El proceso de registro físico es negociable.
Valuable ✅ Libera el inventario para otros usuarios.
Estimable ✅ Flujo sencillo y bien definido.
Small ✅ Acción concreta: registrar devolución.
Testable ✅ Verificable por cambio de estado del libro y cuenta.

Criterios de Aceptación

# CA-001: Devolución en plazo Dado que tengo un libro prestado dentro del período permitido Cuando el bibliotecario registra la devolución en el sistema Entonces el estado del libro cambia a "Disponible" Y el préstamo se marca como "Devuelto" en mi historial
# CA-002: Devolución tardía con multa Dado que devuelvo un libro después de la fecha de vencimiento Cuando el bibliotecario registra la devolución Entonces el sistema calcula la multa según los días de retraso (tarifa configurada) Y notifica al usuario por email con el monto de la multa pendiente
# CA-003: Notificación a usuario en lista de espera Dado que existe un usuario que reservó el libro devuelto Cuando se registra la devolución Entonces el sistema envía automáticamente una notificación al primer usuario en la lista de espera

HU-004: Reserva de Libro No Disponible  |  Media  |  Estimación: 3 puntos

Como usuario de la biblioteca,
quiero reservar un libro que actualmente está prestado,
para ser notificado cuando esté disponible y poder pedirlo primero.

Evaluación INVEST

Criterio Cumple Justificación
Independent ✅ Independiente; depende funcionalmente de HU-002 pero no técnicamente.
Negotiable ✅ El tiempo máximo de reserva activa es negociable.
Valuable ✅ Mejora la experiencia del usuario y la gestión del inventario.
Estimable ✅ Funcionalidad de lista de espera, estimable.
Small ✅ Cubre solo el acto de reservar, no el préstamo en sí.
Testable ✅ Verificable con pruebas de cola de reservas y notificaciones.

Criterios de Aceptación

# CA-001: Reserva registrada exitosamente Dado que el libro "Don Quijote" está actualmente prestado Cuando hago clic en "Reservar" Entonces el sistema registra mi reserva con posición en la lista de espera Y veo el mensaje "Tu reserva fue registrada. Eres el #2 en la lista de espera."
# CA-002: Notificación de disponibilidad Dado que tengo una reserva activa para un libro Cuando ese libro es devuelto Entonces recibo un email indicando que el libro está disponible Y tengo 48 horas para solicitar el préstamo antes de que pase al siguiente en la lista
# CA-003: Cancelación de reserva Dado que tengo una reserva activa Cuando accedo a "Mis reservas" y selecciono "Cancelar reserva" Entonces la reserva se elimina y el siguiente usuario en la lista asciende de posición

HU-005: Renovación de Préstamo  |  Media  |  Estimación: 2 puntos

Como usuario con un préstamo activo,
quiero renovar la fecha de devolución de un libro,
para tener más tiempo de lectura sin tener que devolverlo físicamente.

Evaluación INVEST

Criterio Cumple Justificación
Independent ✅ Independiente; solo requiere un préstamo activo.
Negotiable ✅ El número máximo de renovaciones es negociable.
Valuable ✅ Reduce visitas innecesarias y mejora la experiencia.
Estimable ✅ Lógica simple de extensión de fecha.
Small ✅ Acción puntual: extender fecha de vencimiento.
Testable ✅ Verificable por cambio de fecha en el préstamo.

Criterios de Aceptación

# CA-001: Renovación exitosa Dado que tengo un préstamo activo sin reservas pendientes del libro Y no he alcanzado el límite de renovaciones (máximo 2) Cuando hago clic en "Renovar préstamo" Entonces la fecha de vencimiento se extiende 14 días desde la fecha actual Y recibo un email confirmando la nueva fecha de devolución
# CA-002: No se puede renovar si hay reservas Dado que otro usuario ha reservado el libro que tengo prestado Cuando intento renovar el préstamo Entonces el sistema muestra "No es posible renovar. Hay usuarios esperando este libro."
# CA-003: Límite de renovaciones alcanzado Dado que ya realicé 2 renovaciones del mismo préstamo Cuando intento renovar por tercera vez Entonces el sistema muestra "Has alcanzado el límite de renovaciones permitidas para este libro."
🎓 Conceptos Clave
Concepto Descripción breve
Entrevista semi-estructurada Combina preguntas preparadas con libertad para profundizar. Los 5 tipos: contexto, problema, solución, prioridad, validación.
Regla 80/20 El entrevistado habla el 80% del tiempo; el analista guía con el 20%.
Historia de Usuario Como [rol] / Quiero [acción] / Para [beneficio de negocio]. El "Para" describe valor, no consecuencia técnica.
INVEST Independent · Negotiable · Valuable · Estimable · Small · Testable
Gherkin Dado que [contexto] / Cuando [acción] / Entonces [resultado observable]
Puntos de historia Medida relativa de esfuerzo (no horas). Escala Fibonacci: 1, 2, 3, 5, 8, 13.
