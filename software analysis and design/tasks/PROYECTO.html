Rúbrica de Evaluación por Unidad

📝 Descripción General

La nota final del curso se compone de Prácticas Semanales (40 pts), Participación (10 pts) y el Proyecto Final (50 pts).

A continuación se detalla la distribución de puntos y criterios de evaluación para cada componente.

📊 Rúbrica de evaluación por Unidad
Unidad 1: Fundamentos del Análisis (Semanas 1-2) - Entregable 1 (10 pts)
Criterio Excelente Regular Insuficiente
Identificación de Actores y Alcance
2.5 pts 		(2.5 pts) Claridad en la definición de usuarios y límites del sistema. (1.25 pts) Definición vaga de usuarios o límites difusos. (0 pts) No se identifican actores o alcance.
Historias de Usuario
5.0 pts		 (5.0 pts) Redacción correcta siguiendo INVEST, con criterios de aceptación claros. Mínimo 5 historias. (2.5 pts) Historias mal redactadas o sin criterios de aceptación. (0 pts) No se presentan historias de usuario.
Especificación de Requisitos
2.5 pts		 (2.5 pts) Documento SRS preliminar con estructura base. (1.25 pts) Documento incompleto o sin estructura clara. (0 pts) No se entrega especificación de requisitos.
Unidad 2: Diseño y Arquitectura (Semanas 3-4) - Entregable 2 (10 pts)
Criterio Excelente Regular Insuficiente
Diagrama de Arquitectura
4.0 pts		 (4.0 pts) Representación visual clara de componentes y capas (C4 model o similar). (2.0 pts) Diagrama confuso o con errores en la representación de capas. (0 pts) No se incluye diagrama de arquitectura.
ADRs (Architecture Decision Records)
4.0 pts 		(4.0 pts) Documentación de al menos 2 decisiones clave (ej: Stack, DB) con justificación. (2.0 pts) Documentación incompleta o sin justificación clara. (0 pts) No se documentan decisiones arquitectónicas.
Estructura del Proyecto
2.0 pts		 (2.0 pts) Inicialización correcta del repositorio y estructura de carpetas (scaffolding). (1.0 pts) Estructura desordenada o sin seguir estándares. (0 pts) Repositorio mal inicializado o estructura incorrecta.
Unidad 2: Diseño y Arquitectura (Semanas 3-4) - Entregable 3 (10 pts)
Criterio Excelente Regular Insuficiente
Diagrama de Clases / ER
5.0 pts		 (5.0 pts) Modelo de datos coherente y normalizado. Relaciones correctas. (2.5 pts) Modelo con errores de normalización o relaciones incorrectas. (0 pts) No se incluye diagrama de clases o ER.
Diagramas de Comportamiento
3.0 pts		 (3.0 pts) Diagrama de secuencia o de estados para un flujo crítico (ej: Préstamo). (1.5 pts) Diagrama incompleto o poco claro. (0 pts) No se incluyen diagramas de comportamiento.
Aplicación de Patrones
2.0 pts		 (2.0 pts) Identificación y propuesta de uso de al menos un patrón de diseño. (1.0 pts) Identificación errónea o propuesta no justificada. (0 pts) No se aplican patrones de diseño.
Unidad 3: Implementación y Calidad (Semana 5) - Entregable 4 (10 pts)
Criterio Excelente Regular Insuficiente
Pruebas Unitarias
4.0 pts		 (4.0 pts) Implementación de tests para lógica de negocio crítica (ej: cálculo de multas). (2.0 pts) Tests insuficientes o que no cubren casos críticos. (0 pts) No se implementan pruebas unitarias.
Pipeline CI/CD
3.0 pts		 (3.0 pts) Configuración básica de GitHub Actions (lint, test). (1.5 pts) Configuración incompleta o con errores. (0 pts) No se configura pipeline CI/CD.
Reporte de Calidad
3.0 pts		 (3.0 pts) Análisis estático de código o justificación de cobertura de tests. (1.5 pts) Reporte superficial o sin análisis adecuado. (0 pts) No se entrega reporte de calidad.
2. Participación (10 pts)

Se evalúa la colaboración y asistencia activa durante las sesiones.

  • Asistencia Activa (5 pts): Presencia y participación en discusiones y ejercicios en clase.
  • Code Reviews / Aportes (5 pts): Revisión constructiva del código de compañeros o aportes significativos al repositorio común.
3. Proyecto Final (50 pts)

El proyecto integra todo lo aprendido. Se evalúa el MVP (Producto Mínimo Viable) del Sistema de Biblioteca Digital.

Criterio Excelente Regular Insuficiente
Funcionalidad (MVP)
15 pts		 (15 pts) El sistema permite realizar los flujos principales (Registrar libro, Buscar, Prestar, Devolver). Sin errores críticos. (7.5 pts) El sistema tiene errores menores o falta algún flujo principal. (0 pts) El sistema no funciona o tiene errores críticos.
Arquitectura y Código
15 pts		 (15 pts) Estructura limpia, uso correcto de Next.js/React, separación de responsabilidades, tipos de TypeScript. (7.5 pts) Estructura algo desordenada, mezcla de lógica o tipos incompletos. (0 pts) Código espagueti o mal estructurado.
Documentación
10 pts		 (10 pts) README completo, ADRs finales, diagramas arquitectónicos actualizados, instrucciones de despliegue. (5 pts) Documentación incompleta o desactualizada. (0 pts) Sin documentación.
UI/UX (Vibe Coding)
5 pts		 (5 pts) Interfaz moderna, responsive (shadcn/ui), uso efectivo de componentes. (2.5 pts) Interfaz básica o problemas de diseño responsive. (0 pts) Interfaz pobre o inutilizable.
Despliegue
5 pts		 (5 pts) Aplicación funcional desplegada en Vercel (o similar) y accesible públicamente. (2.5 pts) Despliegue con errores o no accesible siempre. (0 pts) No se ha desplegado la aplicación.
Nota: Para aprobar el curso se requiere una nota mínima de 70/100 y haber entregado el Proyecto Final.
