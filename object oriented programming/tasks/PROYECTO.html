Proyecto Final del Curso (Python)

📝 Instrucciones para el estudiante

Esta actividad consiste en el desarrollo del proyecto integrador final del curso, aplicando todos los conceptos aprendidos.

  • Objetivo: Implementar una aplicación completa utilizando Programación Orientada a Objetos en Python.
Entrega: Sube tu proyecto final a la plataforma.
📊 Rúbrica de evaluación por Unidad
Unidad 1: Fundamentos (10 pts)
Criterio Excelente Regular Insuficiente
Definición de Clases
4.0 pts 		(4.0 pts) Clases con nombres correctos y atributos definidos según requerimientos. (2.0 pts) Clases incompletas o mal nombradas. (0 pts) No se definen clases.
Instanciación y Uso
3.0 pts 		(3.0 pts) Creación correcta de objetos y asignación de valores iniciales. (1.5 pts) Instanciación con errores o sin uso de constructores. (0 pts) No se crean objetos.
Métodos Básicos
3.0 pts 		(3.0 pts) Métodos funcionales que manipulan el estado del objeto correctamente. (1.5 pts) Métodos con lógica errónea o bugs. (0 pts) Sin métodos definidos.
Unidad 2: Encapsulamiento (10 pts)
Criterio Excelente Regular Insuficiente
Control de Acceso
4.0 pts 		(4.0 pts) Uso correcto de private/protected y properties/getters/setters. (2.0 pts) Atributos públicos o encapsulamiento mal aplicado. (0 pts) Todo público sin protección.
Validación de Datos
3.0 pts 		(3.0 pts) Validaciones efectivas en setters/properties para mantener integridad. (1.5 pts) Validaciones débiles o ausentes en datos críticos. (0 pts) Sin validaciones.
Cohesión
3.0 pts 		(3.0 pts) Las clases tienen responsabilidades claras y atributos relacionados. (1.5 pts) Mezcla de responsabilidades en una sola clase. (0 pts) Código espagueti.
Unidad 3: Polimorfismo (10 pts)
Criterio Excelente Regular Insuficiente
Jerarquía de Herencia
4.0 pts 		(4.0 pts) Herencia lógica y bien estructurada (Uso de super(), extends). (2.0 pts) Herencia forzada o mal diseñada. (0 pts) Sin herencia o código duplicado.
Implementación Polimórfica
3.0 pts 		(3.0 pts) Uso de Interfaces/Clases Abstractas (ABC) y métodos abstractos implementados correctamente. (1.5 pts) Implementación parcial o incorrecta de interfaces. (0 pts) Sin uso de polimorfismo.
Sustitución
3.0 pts 		(3.0 pts) Las subclases pueden sustituir a la clase base sin romper el cliente. (1.5 pts) Las subclases rompen la funcionalidad base. (0 pts) Errores en tiempo de ejecución.
Unidad 4: Diseño Avanzado (10 pts)
Criterio Excelente Regular Insuficiente
Principios SOLID
4.0 pts 		(4.0 pts) Aplicación correcta de los principios SOLID (Responsabilidad Única, Abierto/Cerrado, Sustitución de Liskov, Segregación de Interfaces, Inversión de Dependencias) en la refactorización. (2.0 pts) Aplicación de solo 1-2 principios o mal entendimiento. (0 pts) Código monolítico sin SOLID.
Patrones de Diseño
3.0 pts 		(3.0 pts) Implementación clara de patrones (Singleton, Factory, Strategy, etc.). (1.5 pts) Intento de patrón confuso o incorrecto. (0 pts) Sin uso de patrones.
Calidad de Código
3.0 pts 		(3.0 pts) Código limpio, legible y modular, fácil de mantener. (1.5 pts) Código desordenado o difícil de leer. (0 pts) Código no mantenible.
