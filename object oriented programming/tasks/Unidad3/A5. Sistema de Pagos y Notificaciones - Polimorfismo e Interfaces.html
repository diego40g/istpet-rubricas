A5. Sistema de Pagos y Notificaciones — Polimorfismo e Interfaces (Java · C#)

📝 Instrucciones para el estudiante

Esta actividad se centra en el uso de polimorfismo e interfaces para crear sistemas flexibles y extensibles. Elige tu lenguaje (Java o C#) y completa ambas partes.

  • Objetivo: Implementar un sistema de pagos que soporte múltiples métodos (Tarjeta, Efectivo, Transferencia) tratados uniformemente mediante polimorfismo, y un sistema de notificaciones con múltiples canales.
  • Producto esperado:
    • En Java: Paquete model/ con interface MetodoPago e implementaciones PagoTarjeta, PagoEfectivo, PagoTransferencia. interface Notificador con NotificadorEmail y NotificadorSMS. Clase ProcesadorPagos polimórfica. Paquete ui/ con VentanaPagos.java (Swing + CardLayout).
    • En C#: Carpeta Models/ con interface IMetodoPago e implementaciones equivalentes. interface INotificador con implementaciones de Email y SMS. Carpeta Forms/ con FormPagos.cs (WinForms + TabControl).
    • El ProcesadorPagos no debe contener instanceof / is ni if/else por tipo — el polimorfismo debe manejarlo todo.
    • GUI funcional: selector de método, panel dinámico que cambia según el tipo, historial de transacciones y resumen.
Entrega: Sube el código fuente completo (.java o .cs) y una captura de pantalla de la aplicación en ejecución a la plataforma.

1.8 pts 		Lógica condicional (if tipo == 'Visa') en lugar de polimorfismo.
