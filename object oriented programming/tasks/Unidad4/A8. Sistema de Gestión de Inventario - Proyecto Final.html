A8. Sistema de Gestión de Inventario — Proyecto Final (Java · C#)

📝 Instrucciones para el estudiante

Este es el proyecto final integrador del curso. Debes implementar un Sistema de Gestión de Inventario que demuestre todos los pilares de POO junto con una interfaz gráfica completa y persistencia en base de datos.

  • Objetivo: Integrar abstracción, encapsulamiento, herencia y polimorfismo en una aplicación de escritorio real. La clase abstracta Producto es la base; ProductoElectronico y ProductoRopa la extienden. La interface IRepositorio<T> define el contrato CRUD; ProductoDAO / ProductoRepositorio y ProveedorDAO la implementan. La GUI muestra JTabbedPane / TabControl con paneles por entidad.
  • Producto esperado:
    • En Java (IntelliJ IDEA — Maven): model/Producto.java (abstracta, validaciones en setters), ProductoElectronico.java, ProductoRopa.java (herencia), Proveedor.java, Venta.java + ItemVenta.java (composición), interfaces/IRepositorio.java, data/ConexionBD.java (Singleton), data/ProductoDAO.java y data/ProveedorDAO.java (JDBC), ui/VentanaPrincipal.java, ui/PanelProductos.java, ui/PanelProveedores.java.
    • En C# (JetBrains Rider — WinForms .NET 8): Models/Producto.cs (abstracta), ProductoElectronico.cs, ProductoRopa.cs, Proveedor.cs, Venta.cs + ItemVenta.cs, Interfaces/IRepositorio.cs, Data/InventarioContext.cs (EF Core), Data/ProductoRepositorio.cs, Data/ProveedorRepositorio.cs, Forms/FormPrincipal.cs, Forms/FormProductos.cs, Forms/FormProveedores.cs.
    • Script SQL completo con tablas categorias, proveedores, productos, ventas, items_venta y datos de prueba (Java/MySQL).
    • GUI funcional: menú principal, pestañas por módulo, tabla que carga al iniciar, formulario lateral con validaciones, botones CRUD y búsqueda por nombre.
Entrega: Sube el código fuente completo (.java o .cs), el script SQL (Java) o las migraciones EF Core (C#) y una captura de pantalla de la aplicación funcionando con datos reales en la tabla a la plataforma.
