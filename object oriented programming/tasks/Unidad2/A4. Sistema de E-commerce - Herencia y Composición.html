A4. Sistema de E-commerce — Herencia y Composición (Java · C#)

📝 Instrucciones para el estudiante

Esta actividad explora la diferencia entre relaciones "Es un" (Herencia) y "Tiene un" (Composición). Elige tu lenguaje (Java o C#) y completa ambas partes.

  • Objetivo: Diseñar un sistema de E-commerce utilizando clases abstractas para los productos (Herencia) y objetos contenedores para pedidos (Composición). Luego conectarlas a una interfaz gráfica con catálogo, carrito y pedidos.
  • Producto esperado:
    • En Java: Paquete model/ con clase abstract Producto y subclases ProductoFisico, ProductoDigital, Servicio. Clases ItemCarrito, Carrito, Pedido por composición. Paquete ui/ con VentanaEcommerce.java (Swing).
    • En C#: Carpeta Models/ con abstract class Producto y subclases equivalentes. Carpeta Forms/ con FormEcommerce.cs (WinForms).
    • Demostración de polimorfismo: calcularCosto() / CalcularCosto() con comportamiento distinto por tipo.
    • GUI funcional: agregar productos al catálogo, gestionar carrito y generar pedido con ticket.
Entrega: Sube el código fuente completo (.java o .cs) y una captura de pantalla de la aplicación en ejecución a la plataforma.
