📝 A18. Tarea: Implementar dashboard para obtener los datos del local de ZIPKIN y Grafana

🎯 Objetivo de la actividad

Configurar un dashboard funcional en Grafana que consuma datos de origen local generados por ZIPKIN o mediante una fuente de datos simulada, permitiendo visualizar información relevante de trazas o métricas del sistema.

📌 Instrucciones para el estudiante
  • Verifica que ZIPKIN y Grafana estén correctamente instalados y en funcionamiento en tu máquina local.
  • En Grafana, realiza la conexión con una fuente de datos adecuada para representar trazas u observabilidad:
    • Puedes integrar Prometheus, Jaeger o utilizar archivos JSON si es necesario.
    • ZIPKIN no se conecta directamente a Grafana, por lo que puedes utilizar fuentes compatibles (Prometheus, Elasticsearch, InfluxDB, etc.) que reciban datos de ZIPKIN o emulen el comportamiento.
  • Crea un dashboard personalizado en Grafana que muestre al menos 3 paneles diferentes, por ejemplo:
    • Latencia de las solicitudes
    • Tiempos de respuesta por servicio
    • Flujo de trazas
  • Cada panel debe tener:
    • Título claro
    • Métrica o consulta adecuada
    • Visualización correcta (gráfica, tabla, histograma, etc.)
  • Adjunta evidencia del dashboard funcionando con capturas de pantalla.
  • Documenta brevemente cómo lograste conectar la fuente de datos y configurar el dashboard.
  • Formato de entrega: Documento PDF o presentación con capturas, explicaciones técnicas y resultados.
