A6. Redacción Parcial del Documento SRS — IEEE 29148:2018

📋 Descripción

Integrando los artefactos generados en A2 y A5 (requisitos, stakeholders, historias de usuario), construye un Documento SRS parcial del Sistema de Biblioteca Digital siguiendo la estructura del estándar IEEE 29148:2018 (Systems and Software Engineering — Life Cycle Processes — Requirements Engineering). Completa las secciones de Introducción y los Requisitos Funcionales en formato Entrada/Proceso/Salida.

🎯 Objetivos de Aprendizaje
  1. Aplicar la estructura del estándar IEEE 29148:2018 a un documento SRS real.
  2. Escribir la sección de Alcance diferenciando explícitamente qué incluye y qué excluye el sistema.
  3. Construir un glosario técnico sin ambigüedades para toda la audiencia del documento.
  4. Documentar requisitos funcionales en formato Entrada / Proceso / Salida (E/P/S).
  5. Integrar artefactos previos (HU, requisitos de entrevista) en el documento SRS.

Sistema de Biblioteca Digital

Estándar: IEEE 29148:2018  |  Versión: 1.0  |  Fecha: 10 de marzo de 2026  |  Estado: Borrador

1. Propósito

El presente documento describe los requisitos de software para el Sistema de Biblioteca Digital (SBD). Su propósito es establecer, sin ambigüedades, las funcionalidades, restricciones y atributos de calidad del sistema, sirviendo como contrato técnico entre el equipo de desarrollo y los stakeholders.

Audiencias:

Audiencia Qué lee principalmente
Cliente / Gerencia Propósito, alcance y visión general del sistema
Equipo de desarrollo Requisitos funcionales detallados (formato E/P/S)
QA / Testers Requisitos funcionales para derivar casos de prueba
Arquitectos Restricciones del sistema + requisitos no funcionales
2. Alcance

El SBD es una plataforma web para gestionar el catálogo, préstamos, devoluciones, reservas y notificaciones de la biblioteca.

✅ El sistema INCLUYE:

  • Registro y autenticación de usuarios (lectores y bibliotecarios)
  • Búsqueda y consulta del catálogo de libros
  • Gestión de préstamos, devoluciones y renovaciones
  • Sistema de reservas para libros no disponibles
  • Notificaciones automáticas por email
  • Generación de reportes de uso para el administrador
  • Gestión de multas por devoluciones tardías

❌ El sistema NO INCLUYE:

  • Venta de libros ni pagos en línea
  • Gestión de recursos físicos de la biblioteca (infraestructura, mobiliario)
  • Lectura en línea o descarga de contenido digital con derechos de autor
  • Integración con sistemas externos de otras bibliotecas
  • Aplicación móvil nativa (solo web responsiva)

5. Visión General del Sistema

El SBD opera como plataforma web accedida por tres actores principales:

  • Lector: búsqueda del catálogo, solicitud de préstamos, reservas y renovaciones.
  • Bibliotecario: registro de préstamos y devoluciones, gestión del catálogo.
  • Administrador: configuración del sistema y generación de reportes.

El sistema se integra externamente con:

  • Servicio de email (notificaciones automáticas de confirmación, recordatorios y disponibilidad).
  • Base de datos de catálogo de la institución (fuente de datos de libros).
Requisitos Funcionales (Formato E/P/S)
RF-001: Registro de Usuario Alta
Descripción El sistema debe permitir que nuevos usuarios creen una cuenta.
Entrada Nombre completo, email, contraseña (mín. 8 caracteres, ≥1 mayúscula y ≥1 número), número de carné.
Proceso Validar email no duplicado; cifrar contraseña con bcrypt; generar link de verificación y enviarlo al email.
Salida Cuenta creada en estado "Pendiente de verificación"; email de bienvenida con link de activación.
RF-002: Autenticación de Usuario Alta
Descripción El sistema debe permitir el inicio de sesión de usuarios verificados.
Entrada Email y contraseña.
Proceso Verificar credenciales contra la base de datos; bloquear cuenta 15 min tras 5 intentos fallidos consecutivos; registrar sesión con token JWT.
Salida Sesión iniciada; token emitido; redirección al panel del usuario según su rol.
RF-003: Búsqueda de Libros Alta
Descripción El sistema debe permitir buscar libros del catálogo por múltiples criterios.
Entrada Texto de búsqueda (título, autor o categoría); filtros opcionales (disponibilidad, año, idioma).
Proceso Consultar catálogo filtrando por los criterios ingresados; ordenar resultados por relevancia.
Salida Lista paginada de libros con título, autor, categoría, año e indicador de disponibilidad.
RF-004: Solicitud de Préstamo Alta
Descripción El sistema debe permitir a un usuario registrado solicitar el préstamo de un libro disponible.
Entrada Selección de un libro con estado "Disponible" por parte del usuario autenticado.
Proceso Verificar que no haya préstamos vencidos ni se haya alcanzado el límite (3 simultáneos); registrar préstamo con fecha de inicio y vencimiento (14 días); cambiar estado del libro a "Prestado".
Salida Préstamo registrado en el sistema; email de confirmación con título, fecha de inicio y fecha de devolución.
RF-005: Registro de Devolución Alta
Descripción El sistema debe permitir al bibliotecario registrar la devolución de un libro prestado.
Entrada Identificador del préstamo o código de barras del libro.
Proceso Comparar fecha de devolución real con la fecha de vencimiento; calcular multa si hay retraso ($0.50/día); cambiar estado del libro a "Disponible"; notificar al primer usuario en cola de reservas.
Salida Préstamo marcado como "Devuelto"; libro disponible en catálogo; notificación al siguiente en reserva (si aplica); multa registrada en cuenta del usuario (si aplica).
RF-006: Renovación de Préstamo Media
Descripción El sistema debe permitir al usuario extender el período de un préstamo activo.
Entrada Solicitud de renovación desde la sección "Mis préstamos" del usuario autenticado.
Proceso Verificar que el préstamo esté activo y no vencido; verificar que no haya reservas activas del libro; verificar que sea la 1.ª o 2.ª renovación (máximo 2); extender la fecha de vencimiento 14 días desde hoy.
Salida Fecha de vencimiento actualizada; email de confirmación con nueva fecha de devolución.
RF-007: Reserva de Libro No Disponible Media
Descripción El sistema debe permitir reservar un libro actualmente prestado.
Entrada Solicitud de reserva sobre un libro con estado "Prestado" por parte del usuario autenticado.
Proceso Registrar reserva con posición en cola FIFO (primero en llegar, primero en ser atendido); asociar al usuario y al libro; notificar la posición en lista al usuario.
Salida Reserva registrada; mensaje de confirmación con posición en lista de espera.
RF-008: Notificación de Disponibilidad por Reserva Media
Descripción El sistema debe notificar al primer usuario en lista de espera cuando un libro reservado es devuelto.
Entrada Evento de registro de devolución de un libro que tiene reservas activas pendientes.
Proceso Identificar al primer usuario en la cola FIFO; enviar notificación por email; iniciar contador de 48 h para que confirme el préstamo; si no confirma en ese plazo, pasar al siguiente usuario de la cola.
Salida Email de notificación al usuario con información del libro y plazo de 48 h para confirmar.
RF-009: Recordatorio de Devolución Alta
Descripción El sistema debe enviar notificaciones automáticas antes del vencimiento de préstamos activos.
Entrada Ejecución programada diaria (job nocturno a las 08:00).
Proceso Identificar todos los préstamos con fecha de vencimiento en exactamente 3 días o en 1 día; generar y enviar email de recordatorio para cada uno.
Salida Email personalizado con título del libro, fecha de vencimiento y enlace directo a la opción de renovación.
RF-010: Generación de Reportes Media
Descripción El sistema debe permitir al administrador generar reportes de actividad de la biblioteca.
Entrada Selección del tipo de reporte (libros más prestados / préstamos activos / multas pendientes) y rango de fechas.
Proceso Consultar y agregar datos del período seleccionado; calcular estadísticas (totales, promedios, top 10); generar archivo en formato PDF o CSV.
Salida Archivo descargable con el reporte solicitado; opción de envío por email al administrador.
🎓 Conceptos Clave
Concepto Descripción breve
IEEE 29148:2018 Reemplaza a IEEE 830-1998. Introduce el ciclo de vida completo de requisitos: elicitación → análisis → especificación → validación → gestión.
StRS Stakeholder Requirements Specification — necesidades en lenguaje de negocio.
SRS Software Requirements Specification — requisitos técnicos; contrato entre equipo y stakeholders.
Formato E/P/S Cada RF describe su Entrada (qué activa al sistema), Proceso (qué hace internamente) y Salida (qué produce).
Alcance Lista explícita de qué incluye y qué NO incluye el sistema. Previene el scope creep (expansión no controlada del alcance).
Trazabilidad Cada RF del SRS debe estar vinculado a una HU o a un requisito descubierto en la entrevista.
