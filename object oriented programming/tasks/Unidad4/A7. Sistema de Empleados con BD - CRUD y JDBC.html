A7. Sistema de Empleados con BD — CRUD y JDBC (Java · C#)

📝 Instrucciones para el estudiante

Esta actividad integra POO + Bases de Datos + GUI. Conecta una aplicación Java o C# a una base de datos relacional usando JDBC (Java) o ADO.NET / EF Core (C#), implementando el patrón Repository/DAO.

  • Objetivo: Crear un sistema de gestión de empleados con CRUD completo conectado a BD. Aplicar el patrón Repository con una interface IEmpleadoRepositorio / IRepositorio<T>. Interfaz gráfica con JTable / DataGridView + formulario lateral + botones CRUD + búsqueda por nombre.
  • Producto esperado:
    • En Java (IntelliJ IDEA — Maven): Paquetes model/ (Empleado.java con validaciones), interfaces/ (IEmpleadoRepositorio.java), data/ (ConexionBD.java Singleton + EmpleadoRepositorio.java con JDBC), ui/ (VentanaEmpleados.java Swing). Script SQL empresa_db.sql.
    • En C# (JetBrains Rider — WinForms): Carpetas Models/ (Empleado.cs), Interfaces/ (IRepositorio.cs), Data/ (ConexionBD.cs ADO.NET + EmpleadoRepositorio.cs o EmpleadoRepositorioEF.cs EF Core), Forms/ (FormEmpleados.cs).
    • Tabla empleados: id, nombre, cargo, salario, activo. Operaciones: Insertar, ObtenerTodos, ObtenerPorId, Actualizar, Eliminar, BuscarPorNombre.
    • Validaciones GUI: campos vacíos muestran mensaje de advertencia; salario ≤ 0 no permite guardar. Búsqueda por nombre con LIKE / EF.Functions.Like.
    • Sentencias preparadas obligatorias — sin concatenación SQL directa (prevención de inyección SQL).
Entrega: Sube el código fuente completo (.java o .cs), el script SQL y una captura de pantalla de la aplicación en ejecución mostrando la tabla con datos a la plataforma.
