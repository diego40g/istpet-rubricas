A6. Sistema de Gestión con Abstracción — Clases Abstractas y Template Method (Java · C#)

📝 Instrucciones para el estudiante

Esta actividad aplica abstracción mediante clases abstractas y el patrón Template Method. Elige tu lenguaje (Java o C#) y completa ambas partes.

  • Objetivo: Implementar FormularioBase abstracta con un Template Method procesarEnvio() que define el flujo fijo de validación y guardado. Crear subclases para Empleado, Producto y Cliente. Añadir un GeneradorReporte abstracto con Template Method. GUI con panel dinámico por tipo.
  • Producto esperado:
    • En Java: Paquete model/ con abstract class FormularioBase y método final procesarEnvio(). Subclases: FormularioEmpleado, FormularioProducto, FormularioCliente. abstract class GeneradorReporte con subclases ReporteEmpleados y ReporteProductos. Paquete ui/ con VentanaFormularios.java (Swing + CardLayout).
    • En C#: Carpeta Models/ con abstract class FormularioBase, propiedad abstracta Titulo y método sealed ProcesarEnvio(). Mismas subclases equivalentes. Carpeta Forms/ con FormGestor.cs (WinForms + TabControl).
    • Validaciones: Empleado (nombre ≥3, salario ≥$172, email con @), Producto (nombre no vacío, precio >0, stock ≥0), Cliente (nombre ≥3, cédula 10 dígitos, teléfono 9-10 dígitos).
    • GUI funcional: selector de tipo, panel dinámico, log de operaciones, barra de estado con contadores, botón "Generar Reporte".
Entrega: Sube el código fuente completo (.java o .cs) y una captura de pantalla de la aplicación en ejecución a la plataforma.
